# Storyline Guide

The deck is an argument, not a document. Build the argument first.

## Governing thought

One sentence the whole deck exists to prove. If the user can't state it,
help them find it by asking: *what should the audience decide or do after
this meeting?* The governing thought is the answer to the question that
decision raises.

**Interview technique.** When drawing the storyline out of the user, use the
`grilling` technique: ask **one question at a time**, and give **your
recommended answer with each one** ("I'd lead with the cost finding because
it's what forces the decision — agree, or is timeline the real driver?").
Reacting to a proposal surfaces the real takeaway faster than an open
"what's your message?" Resolve one branch before opening the next.

## Pyramid structure

- **Top:** governing thought (becomes the exec summary).
- **Middle:** 3–5 supporting arguments (become section dividers). They must
  be MECE — mutually exclusive, collectively exhaustive. If two sections
  overlap, merge or re-cut them.
- **Bottom:** evidence slides, each proving exactly one point of its section.

Order sections by the audience's logic, not the team's workplan. Two standard
orders:

- **Situation → Complication → Resolution** for recommendations.
- **What happened → Why → What next** for status/readout decks.

## Action titles

Every slide title is a complete sentence asserting the slide's takeaway.

| Weak (topic label) | Strong (action title) |
|---|---|
| Cost analysis | Indirect spend drives 60% of the cost gap to peers |
| Implementation timeline | Wave 1 delivers savings before the FY27 budget cycle |
| Risks | Two risks need steering-committee decisions today |

Test: read only the titles in order. If the story is complete and persuasive,
the storyline works. If a title can't be written as an assertion, the slide
has no message — cut it or push it to appendix.

## The outline deliverable

Before building, produce and confirm:

```
Governing thought: <one sentence>

1. Title slide
2. Exec summary — <governing thought + 3 supports>
3. [Section A] <assertion>
   3.1 <action title> — pattern: <from slide-patterns.md> — evidence: <source>
   3.2 ...
4. [Section B] ...
...
N. Next steps — <who does what by when>
Appendix: <list>
```

Each content line carries its pattern and its evidence source. A line with
no evidence is a flag to resolve before rendering, not after.

## Audience calibration

- **Board / C-suite:** ≤ 15 slides in the main pack, everything else
  appendix. One idea per slide, big numbers, no methodology in the body.
- **Working team:** density is acceptable, but action titles still rule.
- **Steering committee:** decisions needed go in the first three slides,
  flagged explicitly ("Decision required").
